# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead:
1. Open a [GitHub Security Advisory](../../security/advisories/new) on this
   repository (preferred), or
2. Email the maintainers privately with a description of the issue, steps to
   reproduce, and potential impact.

We aim to acknowledge reports within 5 business days and to release a fix or
mitigation within 30 days for confirmed high/critical issues.

## Scope & Known Considerations

This is a research/portfolio demand-forecasting platform with synthetic
data by default. If you deploy it with **real** business data, please note:

- The FastAPI service (`api/`) does **not** ship with authentication enabled
  by default — add an auth layer (API keys, OAuth2, or a gateway like Kong/
  Ambassador) before exposing `/train` or `/retrain` publicly, since these
  endpoints trigger model (re)training and are compute-intensive (potential
  DoS vector).
- The Streamlit dashboard has no built-in access control — front it with
  your SSO/reverse-proxy auth in production (see `deployment/nginx/`).
- `reportlab`/`shap`/`mlflow` and other third-party dependencies should be
  kept up to date — see `.github/workflows/codeql.yml` and
  `requirements.txt` pins; run `pip-audit` periodically.
- Synthetic datasets generated by `src/data/generate_datasets.py` contain no
  real personal or business data. If you swap in real data, review it for
  PII before committing to version control or DVC remotes.
